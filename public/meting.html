api-enhanced 内置的 APlayer / MetingJS 兼容端点 · 完全兼容 injahow/meting-api 协议

简介

这是基于 Meting 与 injahow/meting-api 协议实现的 /meting 端点，运行在 api-enhanced 服务之上。

本站支持解析网易云 VIP 歌曲（通过 Cookie: MUSIC_U=xxx 头传递）和灰色歌曲自动解灰 —— 调用即可使用，无需登录。

GitHub：api-enhanced，本端点基于 Meting 协议构建。

参数说明

server
数据源：netease 网易云音乐（默认）｜tencent QQ 音乐｜xiami 虾米｜kugou 酷狗｜baidu 百度｜kuwo 酷我
（当前 api-enhanced 仅 netease，其他平台静默忽略）
type
name 歌曲名 ｜ artist 歌手 ｜ url 播放链接(302) ｜ pic 封面(302) ｜ lrc 歌词(含中文翻译) ｜ song 单曲JSON ｜ playlist 歌单JSON ｜ search 搜索JSON
id
歌曲/歌单 ID；type=search 时为搜索关键词
br
音质（仅 type=url 生效，默认 320，可选 2000 flac / 192 / 128）
cover
封面分辨率（仅 type=pic 生效，默认 300；兼容旧参数 size）
limit
搜索条数（type=search，默认 30）
page
搜索页码（type=search，默认 1）
search_type
平台搜索类型（默认 1：单曲）
VIP
请求头 Cookie: MUSIC_U=你的token 即可解锁 VIP / 付费歌曲

在线测试

  • 播放链接（302 重定向）: /meting/?type=url&id=33894312
  • FLAC 无损音质: /meting/?type=url&id=416892104&br=2000
  • VIP 歌曲测试（无 cookie 应走解灰）: /meting/?type=url&id=1385117201
  • 封面图（302 重定向）: /meting/?type=pic&id=33894312
  • 封面 500×500: /meting/?type=pic&id=416892104&cover=500
  • 歌词（含中文翻译合并）: /meting/?type=lrc&id=33894312
  • 歌曲名: /meting/?type=name&id=33894312
  • 歌手: /meting/?type=artist&id=33894312
  • 单曲 JSON: /meting/?type=song&id=33894312
  • 歌单 JSON: /meting/?type=playlist&id=2619366284
  • 搜索: /meting/?type=search&id=周杰伦&limit=5
  • server 参数静默忽略: /meting/?type=song&id=33894312&server=tencent

APlayer 集成

<script>
window.meting_api = 'http(s)://你的域名:3002/meting/?server=:server&type=:type&id=:id&auth=:auth&r=:r'
</script>
<meting-js server="netease" type="playlist" id="2619366284"></meting-js>

如需 VIP 支持：在 nginx 等反代层为 /meting/ 注入 Cookie: MUSIC_U=你的token 即可。

与原版 meting 的差异

  • Cookie 完全透传：每次请求的 Cookie 头都会传给网易云（VIP 即时生效），原版 meting 写死了一个固定 cookie。
  • 灰色歌曲自动解灰：底层走 @neteasecloudmusicapienhanced/unblockmusic-utils。
  • 支持随机中国 IP、weapi/eapi/xeapi 多套加密。
  • 仅支持网易云（server=netease），其他 server 值静默忽略。
