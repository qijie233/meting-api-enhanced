常用接口

  • 搜索音乐: GET /search
  • 获取音乐详情: GET /song/detail
  • 获取音乐评论: GET /comment/music
  • 获取音乐播放链接: GET /song/url/v1

🎵 meting 兼容层 新

完全兼容 injahow/meting-api 协议 —— APlayer / MetingJS 客户端可直接指向 /meting 而无需修改前端代码。支持 VIP（Cookie: MUSIC_U=xxx）和灰色歌曲自动解灰。

server
数据源（netease / tencent / xiami / kugou / baidu / kuwo）—— 当前仅 netease，其他平台静默忽略
type
name 歌曲名 · artist 歌手 · url 播放链接(302) · pic 封面(302) · lrc 歌词(含中文翻译) · song 单曲JSON · playlist 歌单JSON · search 搜索JSON
id
歌曲/歌单 ID；type=search 时为搜索关键词
br
音质（type=url 专用，默认 320，可选 2000(flac)/192/128）
cover
封面分辨率（type=pic 专用，默认 300，兼容旧参数 size）
limit
搜索条数（type=search 专用，默认 30）
page
搜索页码（type=search 专用，默认 1）
VIP
请求头 Cookie: MUSIC_U=你的token 即可播放 VIP / 付费歌曲

在线测试

  • 获取播放链接（302 重定向）: GET /meting/?type=url&id=33894312
  • FLAC 无损音质测试: GET /meting/?type=url&id=416892104&br=2000
  • VIP 歌曲测试（无 cookie 应走解灰）: GET /meting/?type=url&id=1385117201
  • 封面图（302 重定向）: GET /meting/?type=pic&id=33894312
  • 封面 500x500: GET /meting/?type=pic&id=416892104&cover=500
  • 歌词（含中文翻译合并）: GET /meting/?type=lrc&id=33894312
  • 歌曲名: GET /meting/?type=name&id=33894312
  • 歌手: GET /meting/?type=artist&id=33894312
  • 单曲 JSON: GET /meting/?type=song&id=33894312
  • 歌单 JSON: GET /meting/?type=playlist&id=2619366284
  • 搜索: GET /meting/?type=search&id=周杰伦&limit=5
  • server 参数静默忽略: GET /meting/?type=song&id=33894312&server=tencent

APlayer 集成示例

<script>
window.meting_api = 'http(s)://你的域名:3002/meting/?server=:server&type=:type&id=:id&auth=:auth&r=:r'
</script>
<meting-js server="netease" type="playlist" id="2619366284"></meting-js>

如需 VIP 支持：在 nginx 等反代层为 /meting/ 注入 Cookie: MUSIC_U=你的token 即可。
